こんにちは、terandard です。

Ruby on Rails 8.1 から Job を中断・再開できる Active Job Continuation という機能が追加されました。この機能を利用すると、長時間実行される Job がデプロイなどによって中断された場合でも、再開して続きから実行することができます。

https://railsguides.jp/active_job_basics.html#%E3%82%B8%E3%83%A7%E3%83%96%E3%81%AE%E7%B6%99%E7%B6%9A

この機能は job-iteration gem から着想を得て実装されたものです。
>This took a lot inspiration from Shopify's job-iteration gem.

https://github.com/rails/rails/pull/55127

弊社のサービスでも job-iteration gem を利用していましたが、Rails 8.1 にアップデートしたので Active Job Continuation に移行しました。

https://zenn.dev/socialplus/articles/1bcab3bc199723

この記事では、job-iteration gem から Active Job Continuation に移行する際のポイントと、Active Job Continuation に移行後に気付いた活用方法について共有します。

## job-iteration gem から Active Job Continuation への移行
### job-iteration gem のサンプルコード

[先述の記事](https://zenn.dev/socialplus/articles/1bcab3bc199723#%E8%A7%A3%E6%B1%BA%E7%AD%96%3A-%E8%90%BD%E3%81%A1%E3%81%A6%E3%82%82%E5%9B%B0%E3%82%89%E3%81%AA%E3%81%84-job-%E3%81%AB%E3%81%99%E3%82%8B-%E2%86%92-job-iteration-gem) に記載している job-iteration gem のサンプルコードは以下のようになっています。

```ruby
class ImportUsersJob < ApplicationJob
  # JobIteration を使う場合、 include する
  include JobIteration::Iteration

  # @param kwargs [Hash] キーワード引数
  # @option kwargs [CsvImport] :csv_import CSV インポートモデル
  # @return [Enumerator, nil] CSV ファイルの行をイテレートする Enumerator
  def build_enumerator(kwargs, cursor:)
    # Job の引数をキーワード引数としている場合は、第1引数にハッシュとして設定される
    csv_import = kwargs.fetch(:csv_import)
    csv_import.file.open do |file|
      # JobIteration の ビルダーを使ってカーソルに従って処理可能にする
      enumerator_builder.csv(CSV.open(file, headers: true), cursor:)
    end
  end

  # @param row [CSV::Row] CSV の行データ
  # @param _kwargs [Hash] キーワード引数
  def each_iteration(row, _kwargs)
    User.create!(
      name: row["name"],
      email: row["email"]
    )
  end
end

# ImportUsersJob.perform_later(csv_import:) で呼び出し可能
```

### Active Job Continuation のサンプルコード
これを Active Job Continuation に移行すると、以下のようになります。

```ruby
class ImportUsersJob < ApplicationJob
  include ActiveJob::Continuable

  def perform(csv_import:)
    @csv_import = csv_import
    step :process_csv, start: 0
  end

  # @param step [ActiveJob::Continuation::Step] カーソル位置を含むステップ情報
  def process_csv(step)
    @csv_import.file.open do |file|
      CSV.open(file, headers: true)
        .lazy
        .drop(step.cursor)
        .each do |row|
          each_iteration(row)
          step.advance!
        end
    end
  end

  # @param row [CSV::Row] CSV の行データ
  def each_iteration(row)
    User.create!(
      name: row["name"],
      email: row["email"]
    )
  end
end
```

### それぞれの違い

job-iteration gem では `build_enumerator` と `each_iteration` を定義する形になっており、慣れるまでは少しわかりづらさを感じるかもしれません。
一方で Active Job Continuation では `perform` メソッド内で `step` ブロックを呼び出す形になっており、通常の Active Job と似たような書き方なので初見でも読みやすさを感じます。

また job-iteration gem ではビルトインの CSV ヘルパーが提供されていましたが、Active Job Continuation ではそういったヘルパーは提供されていないため、CSV を開いたりカーソルの管理は自分で行う必要があります。

## 移行時の注意点

job-iteration gem ではデフォルトで 5 分以上実行される Job は中断・再開するようになっているので、長時間実行され続けることによるリソースの消費を抑えることができます。一方、Active Job Continuation ではそういった制限はないため、特にメモリの使用量に注意する必要があります。
https://github.com/Shopify/job-iteration/blob/61436fde89d110c72f387f32a817c92a29ae7cd1/lib/job-iteration.rb#L24-L48

また job-iteration gem ではエラーが発生した場合は呼び出し元に伝播されるため、自動でリトライされることはありません。一方、Active Job Continuation ではステップを進めた後にエラーが発生した場合には自動でリトライします。`discard_on` でハンドリングしていてもリトライされるので、意図的に実行を停止させる場合は `resume_errors_after_advancing` を `false` に設定する必要があります。
https://github.com/rails/rails/blob/fa8f0812160665bff083a089d2bb2fc1817ea03e/activejob/lib/active_job/continuable.rb#L72-L98

サンプルコードとしては以下のようになります。
```ruby
class ImportUsersJob < ApplicationJob
  include ActiveJob::Continuable

  # step の途中で StandardError が発生した時には、再開せずに失敗させる
  self.resume_errors_after_advancing = false

  # SomeError が発生した場合はリトライせずに実行を停止する
  discard_on SomeError

  def perform(csv_import:)
    @csv_import = csv_import
    step :process_csv, start: 0
  end
end
```

## Active Job Continuation の活用方法

job-iteration gem では単一の処理しか利用できませんでした。
一方で Active Job Continuation では複数の step を定義して、処理の段階ごとにカーソルを管理することもできます。中断・再開時には実行済みの step はスキップされるため、処理の途中で中断されても再開して続きから実行することができます。

これにより長時間実行される Job だけでなく、複数の段階に分かれた Job も中断・再開できるようになりました。
例えば API の呼び出しが必要で冪等性を確保するのが難しい処理なども、ステップごとにカーソルを管理することで中断・再開できるようになります。

```ruby
def perform
  step :process_A
  step :process_B
  step :process_C
end
```

上記の `process_B` の途中で中断された場合、再開すると `process_A` はスキップされて `process_B` の続きから実行されます。Active Job Continuation を利用しない場合は `process_A` の実行状態を管理する必要がありますが、Active Job Continuation を利用することでその管理が不要になります。


## まとめ

job-iteration gem から Active Job Continuation に移行する際のポイントや、Active Job Continuation の活用方法について共有しました。
主な変更点は以下の通りです：

| 項目                   | job-iteration gem                                                | Active Job Continuation                                                  |
| ---------------------- | ---------------------------------------------------------------- | ------------------------------------------------------------------------ |
| include するモジュール | `JobIteration::Iteration`                                        | `ActiveJob::Continuable`                                                 |
| エントリポイント       | `build_enumerator` + `each_iteration`                            | `perform` + `step` ブロック                                              |
| CSV 専用のヘルパー     | `enumerator_builder.csv(csv, cursor:)`                           | なし（手動でカーソル管理）                                               |
| カーソル管理           | `build_enumerator(kwargs, cursor:)` の引数として受け取る         | `step.cursor` で参照、 `step.set!` や `step.advance!` で更新             |
| 開始前処理             | `on_start` コールバック                                          | `perform` 内のステップ前のコード                                         |
| 完了後処理             | `on_complete` コールバック                                       | `perform` 内のステップ後のコード                                         |
| エラー後の再試行       | なし（エラーは呼び出し元へ伝播）                                 | デフォルトは自動リトライする。`resume_errors_after_advancing` で制御可能 |
| Job の実行時間の管理   | `job_iteration_max_job_runtime` で管理（デフォルト `5.minutes`） | なし                                                                     |
| 処理できるステップ数   | 1                                                                | 任意の数                                                                 |
